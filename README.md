# Etalking CRM 增強面板 (Etalking CRM Overlay)

本專案為針對 Etalking 原有 CRM 系統開發之增強面板腳本。透過視覺化數據與動態計算機制，協助業務團隊管理名單，並與 Google Sheets 進行雙向同步。

## 核心功能

### 1. 雙軌制名單跟進追蹤
針對不同生命週期的名單，系統採用不同的追蹤邏輯：
*   **新單 (Type 1)**：
    *   規則：套用「3天6次」的跟進標準。此階段不計算觸及率百分比。
    *   顯示：畫面呈現進度條（如：4/6），供業務確認撥打進度。
*   **常態單 (Type 2)**：
    *   規則：計算動態觸及率 (%)，以追蹤長期的跟進狀況。

### 2. 狀態凍結與防弊機制
當客戶有實質進展時，系統將停止計算觸及率，並將最後一次計算的觸及率寫入 Google Sheet 予以保留：
*   **已接通**：系統自動過濾日誌，若聯絡內容不包含預設的未接前綴詞（如：未接、關機、空號等），即判定為已接通。
*   **預約 Demo**：日誌中出現「預約DEMO」紀錄。
*   **階段判定機制**：為防止業務在新單時期提前紀錄已接通以規避後續稽核，上述免計條件僅在名單轉入常態池 (Type 2) 後的聯絡紀錄才會生效。轉入常態前的聯絡僅計入總次數。

### 3. Google Sheets 同步與資料保護
*   **即時寫入**：業務透過面板新增紀錄時，除了傳送至 CRM 系統，同時會將狀態、評級、備註與最新觸及率同步至 Google Sheet。
*   **名單隔離保護**：針對早期未記錄在 Google Sheet 總表中的常態單，系統僅在面板上計算與顯示觸及率，不會自動為其在 Google Sheet 建立新資料，以維持統計報表的一致性。
*   **批次同步**：提供「同步常態單觸及率」功能，可於背景批次將畫面上所有常態單的最新觸及率更新至 Google Sheet。

## 觸及率計算方式

常態單的觸及率計算公式如下：

*   **起算日**：名單分配日或轉入常態日為第 1 天。
*   **基底目標**：前 3 天的聯絡目標固定為 6 次。
*   **常態目標**：第 4 天起，每天額外增加 1.5 次。
*   **目標次數** = 6 + 無條件進位 ( (持有天數 - 3) × 1.5 )
*   **觸及率** = (目前總聯絡次數 / 目標次數) × 100%

範例說明：
名單持有至第 10 天，目標次數為 17 次 ( 6 + 10.5 無條件進位 )。若業務總共聯絡 10 次，觸及率為 58%。系統將以顏色標示進度落後。

## 開發與建置

專案使用 Vanilla JavaScript 搭配 Vite 進行打包。

1. 安裝依賴套件：
   ```bash
   npm install
   ```

2. 執行建置：
   ```bash
   npm run build
   ```
   建置完成後，輸出的檔案會複製為專案根目錄的 `crm.js`。

## 安裝方式

1. 確認已將打包後的程式碼推送到 GitHub `main` 分支。
2. 在瀏覽器中新增書籤，並將網址列填入以下程式碼：

   ```javascript
   javascript:(function(){
       var script = document.createElement('script');
       script.src = 'https://cdn.jsdelivr.net/gh/JimChen0712/etalking-crm@main/crm.js?v=' + new Date().getTime();
       document.body.appendChild(script);
   })();
   ```

3. 登入 CRM 系統後，點擊該書籤即可啟動面板。

## 專案結構

*   `src/main.js`：主程式（UI 渲染、狀態管理、觸及率與日誌邏輯）。
*   `src/api.js`：Google Apps Script API 通訊模組。
*   `src/config.js`：全域設定檔。
*   `crm.js`：打包後的最終單一執行檔。
